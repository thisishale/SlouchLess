# Requires Python 3.11+ (onnxruntime>=1.24, needed for NumPy 2.x support,
# only ships wheels for 3.11+). On a 3.10-or-older interpreter, pip silently
# resolves an older onnxruntime instead - one whose compiled wheel crashes
# with "AttributeError: _ARRAY_API not found" against NumPy 2.x (a C-API ABI
# break its build predates), even though its metadata loosely allows
# numpy>=1.21.6. NumPy 2.x is required here, not just tolerated: the trained
# .joblib models were pickled under it, and their pickled arrays reference
# numpy._core (renamed from numpy.core in 2.0) - they fail to unpickle
# ("No module named 'numpy._core'") under NumPy 1.x. opencv-contrib-python
# isn't used - nothing here touches its contrib-only modules - so plain
# opencv-python is used instead (already pulled in transitively by rtmlib
# anyway - installing both together is its own source of conflicts).
